JavaScript Comments

Single Line Comments

'//' denotes a single-line comment.

JavaScript will ignore all text between '//' and the end of the line (it will not be executed).

Each code line in this example is preceded by a single-line comment:

                
            
<!DOCTYPE html>
    <html>
        <body>
            <h1 id= "myH"></h1>
            <p id= "myP"></p>
            <script>
 
               // Change heading:
               document.getElementById("myH").innerHTML = "JavaScript Comments"; 

               // Change paragraph:
              document.getElementById("myP").innerHTML = "My first paragraph.";
            <script>
        </body>
    </html>

Output:

                
JavaScript Comments

                
My first paragraph.

Multi-line Comments

Multi-line comments begin with /* and end with */.

JavaScript ignores any text between /* and */.

This example explains the code with a multi-line comment (a comment block):

                        
        
<!DOCTYPE html>
    <html>
        <body>
            <h1 id= "myH"></h1>
            <p id= "myP"></p>
            <script>
                /*
                The code below will change
                the heading with id = "myH"
                and the paragraph with id = "myP"
                in my web page:
                */
                document.getElementById("myH").innerHTML = "My First Page";
                document.getElementById("myP").innerHTML = "My first paragraph.";
            </script>
        </body>
    </html>

Output:

            
            
My First Page

            
My first paragraph.
