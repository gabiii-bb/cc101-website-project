JavaScript Const

In JavaScript, the const keyword is used to declare a variable with a constant value, meaning its value cannot be reassigned after initialization. It provides a way to create variables that are read-only and cannot be changed throughout the execution of the program. However, it's important to note that using const does not make the variable's content itself immutable; it just prevents reassignment of the variable to a different value.

Syntax:

            
                const const_name;
                const x;

Properties:

• Cannot be reassigned.
• It has Block Scope
• It can be assigned to the variable on the declaration line.
• It is a Primitive value.
• The property of a const object can be changed but it cannot be changed to a reference to the new object
• The values inside the const array can be changed, it can add new items to const arrays but it cannot reference a new array.
• Re-declaring of a const variable inside different block scopes is allowed.
• Cannot be Hoisted.
• Creates only read-only references to value.

Example 1: Primitive data type
In this example, we declare a constant variable pi and assign it the value of 3.14. Trying to reassign a new value to pi will result in an error because it's declared with const.

JavaScript const

            
                const pi = 3.14;
                console.log(pi);  // Output: 3.14

                // Attempting to reassign will result in an error
                // pi = 3.14159;  // Error: Assignment to constant variable.

Example 2: Describing the const variable which contains the Block Scope
This JavaScript code illustrates the concept of block scoping using the const keyword. Initially, a constant variable x is declared with the value 22 in the outermost scope. Within a block, a new constant variable x is declared with the value 90, demonstrating how block-scoped variables don't affect the outer scope. Subsequently, two nested blocks introduce their own x variables with values 77 and 45, respectively. Despite these inner declarations, each console.log(x) within the nested blocks outputs their respective local x values. Finally, outside all blocks, a console.log(x) prints the value of the outermost x, which remains unaffected by the inner block declarations, displaying 22. This demonstrates how block scoping confines variable declarations to their respective blocks, preventing unintended variable conflicts.

JavaScript const

            
                const x = 22;
            {
	                const x = 90;
	                console.log(x);

	            {
		                const x = 77;
		                console.log(x);
	            }
	            {
		                const x = 45;
		                console.log(x);
	            }
            }
            console.log(x);

Output:

                           90
                           77
                           45
                           22

Example 3: Object
In this example, we declare a constant variable person and assign it an object with properties name and age. While we can modify the properties of the object, trying to reassign a new object to person will result in an error.

JavaScript const

            
                const person = {
                    name: 'John',
                    age: 30
                  };
                  
                  console.log(person.name);  // Output: John
                  
                  // Modifying the object properties is allowed
                  person.age = 31;
                  console.log(person.age);  // Output: 31
                  
                  // Reassigning the entire object will result in an error

Example 4: Array
Here, we declare a constant variable numbers and assign it an array. Modifying the elements of the array is allowed, but trying to assign a new array to numbers will result in an error.

JavaScript const

            
                const numbers = [1, 2, 3, 4];

                console.log(numbers[0]);  // Output: 1

                // Modifying the array elements is allowed
                numbers[1] = 5;
                console.log(numbers);  // Output: [1, 5, 3, 4]

                // Reassigning the entire array will result in an error
                // numbers = [10, 20, 30];  // Error: Assignment to constant variable.

Output:

                1
                [ 1, 5, 3, 4 ]

Using const is a good practice when you have values that should not be changed throughout the execution of your program. It makes your code more robust by preventing unintentional reassignments.
